## 🚀 Как запустить

### Что понадобится перед стартом

1. **Токен Telegram-бота** — получить у [@BotFather](https://t.me/BotFather):
   отправить `/newbot`, следовать инструкциям, скопировать выданный токен.
2. **Токен Playerok** — значение cookie `token` с сайта playerok.com
   (проще всего достать расширением браузера **Cookie Editor**: зайди на
   playerok.com под своим аккаунтом → открой Cookie Editor → найди cookie
   `token` → скопируй значение).
3. **Python 3.10+** и **git** на Linux-сервере/ПК.

> Свой Telegram ID заранее узнавать не нужно — админом бот делает того, кто
> введёт правильный **пароль ЛК** (см. ниже), а не того, чей ID вписан в конфиг.

---

### Шаг 1. Скачать проект

```bash
git clone <ваш-репозиторий> playerok-vortex
cd playerok-vortex
```

(или просто распакуй присланный `.zip` и зайди в папку через `cd`)

### Шаг 2. Дать права на запуск инсталлера

```bash
chmod +x install.sh
```

### Шаг 3. Установить зависимости

```bash
./install.sh install
```
Скрипт сам создаст виртуальное окружение `.venv` и поставит все нужные
Python-библиотеки.

### Шаг 4. Создать аккаунт (инстанс)

```bash
./install.sh create
```
Скрипт спросит имя инстанса (например `main`) и создаст файл
`instances/main/config.yaml`, а также предложит сразу открыть его в `nano`.

### Шаг 5. Заполнить конфиг (можно пропустить — см. ниже)

Открой `instances/main/config.yaml` и впиши свои данные:

```yaml
telegram:
  token: "12345:AAExxxxxxxxxxxxxxxxxxxxxx"   # токен от @BotFather

playerok:
  token: "..."          # cookie "token" с playerok.com
```

Остальные настройки (авто-восстановление, напоминания, уведомления и т.д.)
можно оставить по умолчанию и донастроить позже прямо из меню бота в Telegram.

> 💡 **Этот шаг можно пропустить.** Если запустить бота вообще без `config.yaml`
> или с незаполненными полями — он сам создаст файл из примера и **прямо в
> консоли спросит по очереди**:
> 1. токен Telegram-бота,
> 2. прокси для Playerok — необязательно, можно пропустить нажатием Enter
>    (форматы: `http://host:port`, `http://логин:пароль@host:port`,
>    `socks5://host:port`, `socks5://логин:пароль@host:port`),
> 3. токен Playerok,
> 4. **пароль доступа к ЛК** — придумай и введи его (только при самом
>    первом запуске, дальше не спрашивается).
>
> Спросит только то, чего реально не хватает — если конфиг уже полный, бот
> стартует молча. Если бот запущен в фоне (например через `./install.sh start`,
> без интерактивного терминала) — не зависнет: сразу выведет в лог, каких
> именно полей не хватает, и завершится, чтобы ты запустил его интерактивно
> хотя бы раз и дозаполнил недостающее.

### Шаг 6. Запустить

```bash
./install.sh start main
```

Если всё верно — бот подключится к Playerok и Telegram и начнёт работать в
фоне. Проверить статус:

```bash
./install.sh list
```

Посмотреть логи, если что-то не запустилось:

```bash
./install.sh
# затем пункт 8 в меню — "Смотреть логи инстанса"
```
или напрямую:
```bash
tail -f logs/main.out.log
```

### Шаг 7. Открыть бота в Telegram

Напиши своему боту `/start`. Он спросит **пароль ЛК** (тот, что ты придумал
на шаге настройки в консоли) — введи его. После верного пароля ты становишься
администратором бота, и следующее `/start` откроет главное меню со всеми
разделами (сделки, чаты, автовыдача, статистика, настройки и т.д.).

> 🔐 Пароль спрашивается у **любого**, кто напишет боту `/start` и ещё не
> является админом — так бот защищён даже без ввода Telegram ID заранее.
> Кто не знает пароль — в меню не попадёт.

---

### Управление ботом (после первого запуска)

```bash
./install.sh stop main       # остановить
./install.sh restart main    # перезапустить (например, после правки конфига)
./install.sh start-all       # запустить все аккаунты сразу
./install.sh stop-all        # остановить все
```

Либо просто запусти `./install.sh` без аргументов — откроется интерактивное
меню со всеми действиями.

### 👥 Мультиаккинг (несколько аккаунтов Playerok)

Каждый Playerok-аккаунт — отдельный "инстанс": свой `config.yaml`, своя
SQLite БД, свой лог, но общий код и venv.

```bash
./install.sh create         # создать ещё один инстанс, например "second"
./install.sh start second
```

### 🛠 Запуск без инсталлера (для разработки)

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp config.example.yaml config.yaml   # заполните токены
python3 main.py config.yaml
```

## 🖥️ Запуск на своём ПК (Windows)

Бота можно гонять прямо на своём компьютере — тогда он использует мощность
именно этого ПК (CPU/RAM/интернет), и ПК должен быть включён и на связи
всё то время, пока бот должен работать.

1. Установи **[Python 3.10+](https://www.python.org/downloads/)** — при
   установке обязательно поставь галочку **"Add Python to PATH"**.
2. Скачай/распакуй проект в любую папку.
3. (При первом запуске) впиши свои данные в `config.yaml` — или пропусти
   этот шаг, бот сам спросит недостающее прямо в консоли (см. выше).
4. Запусти **`run_windows.bat`** двойным кликом (или из консоли:
   `run_windows.bat` / `run_windows.bat instances\main\config.yaml`,
   если конфиг лежит не в корне).

Скрипт сам создаст виртуальное окружение, поставит зависимости (один раз,
при последующих запусках — сразу стартует) и запустит бота в отдельном
окне с приоритетом процесса "выше обычного", чтобы бот не подтормаживал
на слабом ПК, но и не грузил систему как реалтайм-процесс. Чтобы
остановить — закрой открывшееся окно или нажми в нём `Ctrl+C`.

> ⚠️ Если ПК выключится, уйдёт в сон или потеряет интернет — бот
> остановится вместе с ним. Для работы 24/7 используй хостинг/VPS
> (см. ниже), а не домашний ПК.

## 🌐 Запуск на хостинге / VPS (24/7)

Для круглосуточной работы бота — например, чтобы он не выключался вместе
с твоим ПК — заведи недорогой Linux VPS (подойдёт почти любой, даже самый
дешёвый) и разверни бота там через `install.sh` (см. шаги выше — это и
есть инструкция под хостинг) либо напрямую через `main.py`:

```bash
# на хостинге, без install.sh
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
python3 main.py instances/main/config.yaml
```

Чтобы бот продолжал работать после закрытия SSH-сессии, запускай его через
`./install.sh start <имя>` (уже работает в фоне через `nohup`) — либо оберни
`python3 main.py ...` в `systemd`-сервис или `screen`/`tmux`, если хочешь
управлять процессом вручную.

## 🔑 Где взять токен Playerok


Токен и `__ddg5_` — это значения cookie `token` и `__ddg5_` из авторизованной
сессии на playerok.com (DevTools браузера → Application → Cookies).
**Никому не передавайте эти значения** — это полный доступ к аккаунту.

```
  app.py             # центральный объект приложения (BotApp)

  autodelivery.py    # автовыдача / мульти-выдача
  item_ops.py        # авто-восстановление / авто-поднятие лотов
  scheduler.py        # напоминания, авто-подтверждение, автоподнятие по таймеру

```

## 🛒 Магазин плагинов (бесплатные и платные)

В боте есть встроенный магазин плагинов: **🧩 Плагины → 🛒 Магазин плагинов**.

- Каталог — обычный JSON-файл со списком плагинов (пример: `plugins_catalog.example.json`),
  ссылку на него публикует канал **@Vortexplayrock** — впиши её в `marketplace.catalog_url`
  в своём `config.yaml`.
- **Бесплатные** плагины ставятся в один клик прямо из бота.
- **Платные** плагины требуют лицензионный ключ — его выдают после оплаты
  (пиши **@Iucdip**), ключ вводится прямо в диалоге с ботом и проверяется
  автоматически (`core/licensing.py`).
- Владелец канала генерирует ключи покупателям через
  `python3 tools/generate_license.py <plugin_id> [--days 30]`
  (только у себя, секрет из `marketplace.license_secret` — никому не передавать).
- Также доступно ручное управление: `/plugins` в боте, установка `.py`-файлов
  плагинов напрямую в папку `plugins/` (см. `plugins/example_plugin.py`).

## 🆙 Обновление бота (команда /update)

Бот умеет обновляться сам, прямо из Telegram, если он развёрнут из git-репозитория:

- Команда **`/update`** (или кнопка **🆙 Проверить обновления** в разделе
  «🔧 Настройки») проверяет, есть ли на GitHub новая версия
- Если есть — бот покажет, с какого коммита на какой обновится, и попросит
  подтверждение
- После подтверждения: `git pull` → переустановка зависимостей →
  автоматический перезапуск процесса
- **Данные не теряются**: `config.yaml`, база данных (`data/`), логи
  (`logs/`) не отслеживаются git'ом (см. `.gitignore`) и обновлением кода
  никак не затрагиваются — обновляется только сам код бота
- Если что-то пошло не так при установке зависимостей — бот автоматически
  откатывает код обратно на предыдущую рабочую версию

**Требования для работы `/update`:**
1. Бот должен быть развёрнут как git-репозиторий (склонирован `git clone`,
   либо распакован вместе с папкой `.git` — она уже есть в готовом архиве)
2. Должен быть подключён `origin` на твой GitHub-репозиторий:
   ```bash
   git remote add origin https://github.com/ТВОЙ_АККАУНТ/ТВОЙ_РЕПО.git
   git push -u origin main
   ```
3. На сервере должен быть установлен `git` (`apt install git`, если вдруг нет)

Дальше — просто пушь новый код в свой репозиторий (`git push`), и на сервере
жми `/update` в боте.

## 🔒 Безопасность

- Никогда не публикуйте `config.yaml` (там токены) — он в `.gitignore`.
- Никогда не вставляйте токены (Telegram/GitHub/Playerok) в чаты с ИИ, чаты
  поддержки и т.п. Если токен всё же "засветился" — немедленно отзовите его.
- Для GitHub используйте `git push` со своей машины с личным токеном,
  который никому не показывается.
